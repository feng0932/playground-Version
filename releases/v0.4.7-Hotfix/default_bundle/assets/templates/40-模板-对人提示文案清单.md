# 对人提示文案清单

## 0. 使用规则

本文件是安装后入口、总控派发、子线程返回、读取回执和异常控制动作的对人提示文案真源。

使用本文件时必须遵守：

1. 不得临时让大模型拼凑文字。
2. 不得把机器字段、回执字段或 dispatch evidence 暴露到 human-visible reply body。
3. `00 / 01 / 10` 的对人正文默认固定为 `3` 句：
   - 第一句以 `当前由` 或 `当前` 开头；
   - 第二句以 `这轮已确认` 或 `这轮唯一缺口` 开头；
   - 第三句以 `下一步请` 开头。
4. 非对人窗口只允许输出固定一句：`当前不是对人窗口。`
5. 花括号 `{...}` 是运行时或 agent 可替换的占位符；替换时只填人能理解的信息，不填机器字段名。

## 1. 安装结束后，对人的提示文字

### 1.1 CLI 成功提示

```text
[ai-team] install: ok

Next prompt:
{startup_prompt}
```

### 1.2 startup prompt 中的正式入口提示

```text
请接管当前项目根目录 `{project_root}`。
当前固定 prompt 真源：`{prompt_source}`。
安装后最小正式入口动作是 `/总控`。
若当前宿主支持执行命令，你必须先执行 `ai-team runtime --action total_control_entry`，并只消费它输出的唯一合法 opening / host-native dispatch request；不得跳过 resolver / builder 链自行代演 `00 / 01 / 10`。
```

## 2. `00` 派发 `01 / 10` 的对人提示文字

### 2.1 派发 `01-编排-初始化项目包`

```text
当前由总控完成路由判断。
这轮已确认需要进入 `01-编排-初始化项目包`，先补齐项目级基线。
下一步请在新打开的 `01` 子线程继续对话；该线程结束后输入 `/返回总控`，再回到总控输入 `/读取回执`。
```

### 2.2 派发 `10-执行-产品专家`

```text
当前由总控完成路由判断。
这轮已确认项目包基线已可进入产品真源收敛。
下一步请在新打开的 `10-执行-产品专家` 子线程继续对话；该线程结束后输入 `/返回总控`，再回到总控输入 `/读取回执`。
```

### 2.3 子线程启动提示

```text
你现在接管 `{target_agent}`。项目根目录是 `{project_root}`；先读取 `{dispatch_prompt_absolute_path}` 的完整内容，并只按这份正式派发文件执行。
```

## 3. `01 / 10` 输入 `/返回总控` 后的对人提示文字

方案 C 固定口径：

- 有真实 hidden/file carrier 时，机器回执优先写入 hidden/file carrier 或 receipt file。
- 没有真实 hidden/file carrier 时，必须在三句对人摘要之后追加 terminal machine receipt appendix。
- terminal machine receipt appendix 必须包含 `## 工单回执` 与 `## dispatch_evidence`。
- appendix 不属于普通 human-visible 3 句正文；普通三句正文仍只写当前结果、唯一缺口和下一步。

### 3.1 `01` 返回总控

```text
当前由 `01-编排-初始化项目包` 结束本轮项目包窗口，并把阶段回执交还总控。
这轮已确认 `{本轮项目包阶段结果或唯一缺口}`。
下一步请回到 `00-编排-总控` 主线程输入 `/读取回执`，由总控读取回执并判断下一跳。
```

### 3.2 `10` 返回总控

```text
当前由 `10-执行-产品专家` 结束本轮产品窗口，并把阶段回执交还总控。
这轮已确认 `{本轮产品阶段结果或唯一缺口}`。
下一步请回到 `00-编排-总控` 主线程输入 `/读取回执`，由总控读取回执并判断下一跳。
```

## 4. `00` 输入 `/读取回执` 后的对人提示文字

### 4.1 读取成功

```text
当前由总控读取子线程回执并重新判断下一跳。
这轮已确认 `{child_thread}` 已返回 `{return_reason}`，下一步应进入 `{next_target}`。
下一步请在新打开的 `{next_target}` 子线程继续；如果没有打开，请等待总控完成正式派发。
```

### 4.2 读取失败或缺回执

```text
当前不能读取回执。
这轮唯一缺口是没有可消费的 `01 / 10` 正式阶段回执，或回执字段不完整。
下一步请先回到对应子线程输入 `/返回总控`，再回到总控输入 `/读取回执`。
```

## 5. 其他情况下，对人的提示文字

### 5.1 非对人窗口被打开

```text
当前不是对人窗口。
```

### 5.2 在 `00` 主线程误输入 `/返回总控`

```text
当前不能在总控主线程使用 `/返回总控`。
这轮唯一缺口是 `/返回总控` 只属于 `01 / 10` 子线程退场动作。
下一步请在总控继续使用 `/总控` 或 `/读取回执`；如果你还在子线程，请先回到对应子线程输入 `/返回总控`。
```

### 5.3 在 `01 / 10` 子线程误输入 `/读取回执`

```text
当前不能在子线程读取回执。
这轮唯一缺口是 `/读取回执` 只属于 `00-编排-总控` 主线程。
下一步请先在当前子线程输入 `/返回总控`，再回到总控主线程输入 `/读取回执`。
```

### 5.4 在 `01 / 10` 子线程误输入 `/总控`

```text
当前不能在子线程使用 `/总控` 作为退场动作。
这轮唯一缺口是子线程必须先用 `/返回总控` 生成退出快照和回执。
下一步请在当前子线程输入 `/返回总控`，再回到总控主线程输入 `/读取回执`。
```
