# 中国市场 App 上架与个人信息合规检查清单

## 目标

沉淀一套面向中国市场的 App 上架产品合规检查项，供技能在分析 PRD、产品方案、提测包、上架资料时使用。

检查目标覆盖：

1. 是否满足国内主要应用市场上架必须遵循的规则
2. 是否符合中国个人信息收集、使用、共享、删除、注销等监管要求
3. 是否覆盖 `Android`、`HarmonyOS / 华为应用市场`、`iOS / App Store`

## 适用平台

1. `iOS / App Store`
2. `HarmonyOS / 华为应用市场 AppGallery`
3. `国内主流 Android 市场`
4. `Android 自有官网分发场景`

## 使用说明

建议技能对每个检查项统一输出以下字段：

- `检查项`
- `检查目标`
- `适用平台`
- `触发条件`
- `检查方法`
- `判定标准`
- `风险等级`
- `整改建议`
- `依据`

风险等级建议统一为：

- `阻断项`：不满足时，不应提审、不应上架、或高概率被下架
- `高风险项`：不满足时，审核高概率失败，或存在明显监管风险
- `提示项`：非绝对阻断，但建议补齐，否则影响审核通过率或后续巡检

## 法规与规则口径

### 一、现行强制依据

1. `APP 备案相关要求`
2. `中华人民共和国个人信息保护法`
3. `常见类型移动互联网应用程序必要个人信息范围规定`
4. `App 违法违规收集使用个人信息行为认定方法`
5. `网络数据安全管理条例`
6. `儿童个人信息网络保护规定`
7. 各应用市场官方审核指南、开发者协议、资质要求、隐私与权限要求

### 二、前瞻检查项

`互联网应用程序个人信息收集使用规定` 截至 `2026-03-22` 仍应按征求意见稿口径视为前瞻性高风险检查项，不应误判为已正式生效的现行法。

## 技能建议判定逻辑

### 一、直接判定为不应上架

满足以下任一情况时，技能应直接输出 `不建议提审/不建议上架`：

1. 未完成 APP 备案
2. 缺少必须行业资质
3. 隐私政策缺失或内容明显不完整
4. 首次启动即超范围申请高敏权限
5. 收集信息明显超出最小必要范围
6. 不提供账号注销或删除路径
7. 元数据与实际功能明显不符
8. 存在恶意代码、静默下载、规避商店审核等行为

### 二、判定为高风险整改后再提审

满足以下任一情况时，技能应输出 `建议整改后再提审`：

1. 第三方 SDK 未完整披露
2. 权限使用说明不清晰
3. 拒绝非必要权限后核心流程异常
4. 自动续费、会员、支付说明不充分
5. 审核账号、审核说明、测试数据不完整
6. 未成年人场景、UGC 场景、AI 场景缺少治理策略
