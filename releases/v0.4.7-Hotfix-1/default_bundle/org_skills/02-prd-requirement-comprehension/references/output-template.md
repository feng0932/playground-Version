# 标准输出模板

## 使用说明

- 将标题替换为具体项目名，例如：`<项目名>需求理解分析`。
- 不要在正式输出中保留本模板的说明文字。
- 保持章节和编号顺序稳定。
- 如果某一章节当前材料不足以支撑判断，不要删除该章节；明确写出“当前材料不足以支持判断”。
- 所有关键判断优先使用表格表达。
- 所有关键“缺失 / 冲突 / 语义重叠 / 边界混用”都要附证据。
- 证据格式统一为 `文档名:行号`。

## 模板使用提醒

- 这是一份通用模板，不要把某个项目的事实直接固化到模板里。
- 标准化的是章节、判断方法、表格列和证据写法，不是具体结论。
- 如果本次分析范围只覆盖局部模块，标题和结论中要明确写出局部范围，不要暗示覆盖全量产品。
