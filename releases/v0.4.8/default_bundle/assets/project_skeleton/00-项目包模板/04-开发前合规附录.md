# 开发前合规附录

## 1. 文件用途

这份文件回答：

- 当前项目的协议与隐私专题先读哪里
- 当前项目是否命中实名专题，以及实名如何切边界
- 当前项目是否命中用户治理专题，以及治理承诺级别是什么
- 后续 `10-执行-产品专家`、`21-审查-产品审查`、原型链和模块 `PRD` 该读哪个项目级入口

它是项目级合规专题入口，不替代模块级 `PRD / 4.2A / 4.3 / 4.4`。

## 2. 当前项目协议与隐私入口

这部分回答：

- 首次启动同意机制、登录前协议勾选、第三方服务商协议确认是否命中
- 权限申请是否按功能触发
- 第三方能力与共享口径由谁冻结、谁展示、谁引用
- 用户权利、撤回同意、隐私投诉与账号注销如何切边界

若本节未形成，命中个人信息处理、权限、第三方共享、隐私投诉、撤回同意等场景时，不得继续放行设计或原型前审查。

## 8. 联动专题入口要求

- 只要讨论登录、注册、账号状态主链、游客边界、登录后回跳与来源恢复，就必须回链 `当前项目账号基线入口`
- 只要讨论个人信息处理、权限、第三方共享、撤回同意、隐私投诉，就必须回链 `当前项目协议与隐私入口`
- 只要讨论实名目的、实名信息、实名失败、实名删除边界，就必须回链 `当前项目实名入口`
- 只要讨论冻结、封禁、申诉、人工治理、内容治理、AIGC 治理，就必须回链 `当前项目用户治理入口`
