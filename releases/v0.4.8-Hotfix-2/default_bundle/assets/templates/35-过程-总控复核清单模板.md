# 总控复核清单模板
## 1. 使用规则
- `35-过程-总控复核清单.md` 只做 review-phase echo。
- `当前责任模式 / 模式来源 / task_chain_id / task_chain_epoch / 当前状态 / 是否已完成二次握手` 必须逐字镜像 authority carrier 当前快照，不得自行推断。
- `return_target` 只表示 business-consumer contract；`same_stage_return_targets` 只服务 same-stage remediation semantics；总控只允许在 `/读取回执` 后依据正式回执重判下一步。

## 3. 深审前总控门禁
- 本章各表用于汇总既有审查结论与覆盖情况，不要求总控脱离 `21 / 22 / 23 / 25` 单独重做实体审查。
- 若缺少可回链证据，总控应记录为 `未覆盖 / 需补审 / 不通过`，而不是自行补判业务结论。

- 首波 `routing blocker code` 固定为 `project_compliance_prereview_missing / topic_baseline_missing / module_split_confirmation_missing / prd_formalization_missing / prototype_entry_missing`；若命中其一，`下一跳请求` 只能写同阶段补料动作，不得写继续推进。
