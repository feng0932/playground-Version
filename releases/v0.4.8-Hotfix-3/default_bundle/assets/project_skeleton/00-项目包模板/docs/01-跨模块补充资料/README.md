# 跨模块补充资料目录说明

这里是根 `docs/` 的白名单补充区，不是事实源。

只有满足以下条件的材料才允许写在这里：

1. 非事实源
2. 跨多个模块
3. 已明确批准长期保留

本目录必须再分成两层：

- `local/` 只允许本地保留，不进入远端 git
- `published/` 才允许进入远端 git

每份写入本目录的文档，头部至少需要说明：

- 这不是事实源
- 为什么必须跨模块保留
- 谁批准放这里
- 对应的正式事实源文件有哪些

如果资料能明确归属到项目级或模块级推进过程，应该优先写回：

- `00-项目包/04-过程文件/`
- `01-模块执行包/<模块名>/04-过程文件/`

`记录同步 / 冲突追因 / 复盘 / 审计` 可以显式读取 `00-沟通记录与结论回写.md`，但执行类 Agent 默认不以这里或 `04-过程文件/` 作为主上下文。
