# 根 docs 说明

这份文件用于说明真实项目仓库根 `docs/` 的白名单边界。

## 1. 角色定位

- 根 `docs/` 不是事实源。
- 根 `docs/` 只允许放“非事实源、跨模块、明确批准”的补充资料。
- 一旦材料可以明确归属到项目级或模块级事实源，必须回写到 `00-项目包/` 或 `01-模块执行包/`，而不是长期停留在根 `docs/`。

## 2. 当前允许的白名单目录

- `docs/00-说明.md`
- `docs/01-跨模块补充资料/README.md`
- `docs/01-跨模块补充资料/local/`
- `docs/01-跨模块补充资料/published/`

除此之外，不得默认派生 `docs/superpowers/`、`docs/plans/`、`docs/specs/` 或其他未经批准的根 `docs/` 目录。

## 3. 使用规则

- 如果一时无法判断归属，默认先写入 `00-项目包/04-过程文件/`。
- 只有当资料确实跨模块、非事实源、且明确批准长期保留时，才允许进入 `docs/01-跨模块补充资料/`。
- `docs/01-跨模块补充资料/local/` 只允许本地保留，不进入远端 git。
- `docs/01-跨模块补充资料/published/` 只有明确批准后，才允许进入远端 git。
- 执行类 Agent 默认不读取 `04-过程文件/`，避免把过程痕迹当成事实源或主上下文。
- `总控 / 门禁 / 四审 / 收口` 可以显式读取 `01-总控复核清单.md`。
