# UI/UE 高保真非闭合交回最小要求

| 项目 | 填写内容 |
|------|------|
| 文档角色 | UI/UE 首份高保真交回的最小人读要求 |
| 适用阶段 | `UI/UE执行与产品收回` 的非闭合状态 |
| 使用方式 | 先拿回交付，验证方法论能否消费，再和 UI/UE 团队补齐接口契约 |

## 1. 交回状态

- 本模板用于非闭合状态，不代表产品已验收通过。
- UI/UE 团队只需交回最小可消费材料。
- 产品确认清单由 `10-执行-产品专家` 负责，不要求 UI/UE 团队代替产品拍板。

## 2. 必须交回的 3 份内容

| 交付物 | 最低要求 |
|------|------|
| 可打开的高保真包 | Figma 链接、可运行 HTML 包、可运行前端代码包或可预览原型链接；如果需要运行，写清楚运行方式。 |
| 页面清单 | 列出本次做了哪些模块、页面、入口 / 路由、文件或链接。 |
| 与 PRD 不一致的地方 | 列出 UI/UE 已知的主动调整、不确定点或可能与 PRD 不一致的地方。 |

## 5. 方法论收回口径

- 总控可先按非闭合状态接收这 3 份材料。
- `10-执行-产品专家` 基于交回材料生成产品确认清单、PRD 差异处理建议和一致性验证草案。
- 缺少任一最小交付物时，只能判定为 `需补 UI/UE`，不得外推为模块级绿灯或整包可放行。
